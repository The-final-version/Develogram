조회하는 기능부터 테스트를 하는것이 좋아보인다.

작성, 삭제의 검증 과정에서 조회를 하기 때문이다.
근데 조회를 하려면 작성을 해야하므로 작성과 조회를 처음에 하는것으로 정한다.

시나리오 :

0. 사용자 A가 포스트 X를 작성한다.

1. 포스트에 댓글 작성 후 댓글 ID로 댓글 조회하기.
	사용자 B가 포스트 X에 댓글을 작성한다.
	201을 보내줌.
    댓글 ID로 조회 시 해당 댓글이 반환되며 200을 보내줌
2. 존재하지 않는 댓글 ID로 조회하기
    잘못된 댓글 ID로 조회 시 적절한 오류 메세지와 404를 보내줌
3. 포스트 ID로 댓글 목록 조회하기
    사용자 C, D가 포스트 X에 댓글을 작성한다.
    포스트 ID로 조회 시 댓글의 리스트가 반환되며 200을 보내줌.
    리스트에는 B, C, D가 작성한 댓글이 있다.
4. 존재하지 않는 포스트 ID로 댓글 목록 조회하기
    잘못된 포스트 ID로 조회 시 적절한 오류 메세지와 404를 보내줌

(여기까지 조회 테스트)

5. 작성한 댓글 삭제하기
    사용자 B가 자신이 작성한 댓글을 삭제한다.
    204를 보내줌.
    댓글 ID로 조회하면 오류메세지와 404에러.
    포스트 ID로 조회하면 조회되는 리스트에 B의 댓글이 포함되어 있지 않다.
6. 포스트 작성자가 댓글 삭제하기
    사용자 A가 포스트 X에 작성된 사용자 C의 댓글을 삭제한다.
    204를 보내줌.
    댓글 ID로 조회하면 오류메세지와 404에러.
    포스트 ID로 조회하면 조회되는 리스트에 C의 댓글이 포함되어 있지 않다.
7. 작성 실패
    사용자 B가 빈 댓글을 작성한다.
    400을 보내줌
    포스트 ID로 조회 시 A의 댓글이 없다.
8. 권한이 없는 삭제 요청
    사용자 C가 사용자 D가 작성한 댓글을 삭제하려고 한다.
    403을 보내줌
    댓글 ID를 조회하면 D가 작성한 댓글이 나온다.
    포스트 ID로 조회하면 조회되는 리스트에 D의 댓글이 포함되어 있다.
9. 인증되지 않은 사용자
    로그인 되지 않은 사용자가 D의 댓글을 삭제하려고 한다.
    401을 보내줌
    댓글 ID를 조회하면 D가 작성한 댓글이 나온다.
    포스트 ID로 조회하면 조회되는 리스트에 D의 댓글이 포함되어 있다.
